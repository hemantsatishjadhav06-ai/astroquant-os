# Document 003 — System Architecture (detail)

See `000` §2 for the 8-layer diagram. This document fixes service boundaries and the data-flow contract.

## Service boundaries
- **Collectors (1–5)** are stateless workers; their only side-effect is upserting into the warehouse.
- **Feature Builder (6)** reads warehouse → writes feature store. Pure, versioned, point-in-time.
- **Research (7–8)** reads feature store → writes hypotheses/signals/discoveries.
- **Backtester (9)** reads strategy specs + feature store + bars → writes backtests.
- **Paper (11)** reads signals + live/replay data → writes paper_* + attribution.
- **Cross-cutting (10,13,14,15,0)** observe and govern; never sit in the data hot path except risk gating in paper.

## Data-flow contract
1. Agents communicate only via DB + Redis Streams events (`004`), never direct calls.
2. Everything idempotent; re-runs heal, never duplicate.
3. Every write is provenance-stamped (run_id, code_commit, dataset_hash, seeds).
4. Gates (`000` §7) are enforced by the Orchestrator: a red gate blocks downstream wiring.

## Deployment (v1, single box or small cluster)
`docker-compose`: postgres+timescale, mongodb, redis, (pgvector in postgres), the python services,
and n8n. C++ components built as shared libs (`pybind11`) consumed by the Python collectors/engines.
Secrets via `.env`/pydantic-settings. Backups: nightly Postgres dump + Mongo dump; feature-store Parquet versioned.

# Document 008 — Feature Factory

**Goal:** for every `(date, symbol)`, emit 1000+ point-in-time-correct, family-tagged features.
Built with polars; each feature has a versioned definition in `feature_catalog`.

## Families & examples
- **technical:** returns (multi-horizon), RSI, MACD, ATR, VWAP, Bollinger, volume delta, realized vol, momentum, mean-reversion z-scores, gap stats, candlestick encodings, rolling skew/kurtosis.
- **market:** sector strength/relative-strength, breadth, index correlation/beta, options OI, PCR (OI & vol), max-pain distance, ATM IV, IV skew, VIX level/term, FII/DII flow (if sourced).
- **astro:** per-body sidereal longitude (sin/cos encoded to respect circularity), sign, nakshatra (one-hot or embedding), retrograde flags, moon phase/tithi, combustion, pairwise aspect angles (continuous), dasha lord indicators.
- **gann:** day-count since pivot, active cycle indicators (45/90/180/360/720 proximity), distance to nearest Square-of-Nine level, Gann-angle position, price-time squaring proximity.
- **news:** net sentiment, sentiment momentum/surprise, coverage volume, dispersion (all lagged to publication).
- **macro:** levels & changes of CPI/WPI, rates, USDINR, crude, gold, yields, DXY — joined on `release_date<=as_of`.

## Hard rules
- **Circular encoding** for angles/longitudes: use `sin(θ), cos(θ)`, never raw degrees as linear input.
- **No look-ahead:** every feature computable from `ts<=as_of`. Leakage tests per `007` §6.
- **Versioning:** changing a definition creates a new `feature_id`/version; never silently mutate.
- **Family tag is mandatory** — it powers the baseline/augmented ablation (RQ-004).

# Document 009 — AI & ML

## Model menu
- **Tabular (primary):** XGBoost, LightGBM, CatBoost — strong on heterogeneous features; native handling of missing/categoricals (CatBoost for nakshatra/sign categoricals).
- **Sequence:** LSTM, Temporal Fusion Transformer — for multi-horizon, multi-series temporal structure.
- **Always include a naive baseline** (random / buy-hold / last-sign) — nothing counts unless it beats naive out-of-sample.

## Tasks
- Classification: "NIFTY up tomorrow?", "BANKNIFTY moves >1%?", "stock outperforms sector?"
- Regression: next-horizon return; IC is the headline metric.

## The ablation harness (how RQ-004 is mechanized)
- Train **baseline** (technical∪market) and **augmented** (+astro+gann) with identical CV/walk-forward.
- Compare out-of-sample AUC/IC and resulting strategy DSR.
- **SHAP** for attribution; **permutation importance** as cross-check; **random-feature control** must rank near-zero.
- Penalize complexity: augmented must beat baseline by more than noise, after correction (`007` §4).

## Validation & robustness
- Purged/embargoed walk-forward only; never random shuffle.
- Hyperparameter search counts toward the trial denominator (feeds DSR/PBO).
- **Drift detection:** monitor feature/label distribution shift (PSI/KL); retrain triggers logged; concept drift demotes signals.

## LLM role
Narration, hypothesis generation, sentiment — never the significance verdict. Stats code decides; LLM explains.

# Document 010 — Backtesting

## Two engines, two jobs
- **VectorBT (vectorized):** fast parameter sweeps during *discovery*. Cheap, approximate. Every sweep increments the trial counter.
- **C++ event-driven engine:** honest, path-dependent backtests before paper. Models order→fill latency, partial fills, realistic costs (shared cost model with `011`). This is what a strategy must pass.

## Methods (all mandatory before G4)
- **Event-driven** simulation with the same cost/fill model as paper (`011` §3) — consistency matters.
- **Walk-forward** out-of-sample evaluation (rolling train/test).
- **Monte Carlo:** resample trade sequences / bootstrap returns to get performance confidence intervals (not a single lucky path).
- **Regime testing:** bull / bear / high-vol / low-vol / crisis splits; report per-regime metrics.
- **Deflated Sharpe Ratio + PBO** reported for any strategy chosen from a search (`007` §4).

## Metrics
Win rate, Sharpe, Sortino, max drawdown, profit factor, Calmar, turnover, **DSR**, **PBO** — all **post-cost**.

## Anti-overfitting guards
- Out-of-sample is sacred; test set touched once.
- Trial count from VectorBT sweeps + hyperparam search flows into DSR.
- A backtest result is meaningless without its dataset_hash + code_commit + trial count.

# Document 012 — Risk Management

Applies in backtest and paper (and would gate any future live step). Implemented by Agent 10.

## Layers of risk
- **Position risk:** max size per symbol (as % equity and vs. recent ADV), per-trade stop logic, max leverage.
- **Portfolio risk:** gross/net exposure caps, per-sector concentration limits, correlation-aware sizing, portfolio max-drawdown circuit breaker (halt new entries on breach).
- **Strategy risk:** per-strategy capital allocation, per-strategy drawdown kill-switch, decay demotion (Performance Auditor).
- **System risk:** data-feed failure handling, stale-data guards (no trading on stale quotes), reconciliation invariants (`011` §4), kill-switch on anomaly.
- **Black-swan / tail:** scenario stress (gap-down, vol spike, liquidity evaporation), position limits sized to survive fat tails, explicit assumption that historical vol understates tail risk.

## Pre-trade gate
Every intended order: check position/portfolio/strategy/system limits → **approve / scale / reject**, logged with reason, surfaced in reports. Rejections are data (they reveal strategy aggressiveness).

## Reporting
Daily risk snapshot: exposures, concentration, drawdown vs limits, any breaches/vetoes, tail-scenario estimates.
